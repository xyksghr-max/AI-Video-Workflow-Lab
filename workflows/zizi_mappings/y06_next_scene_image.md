# Y06 字字动画映射卡: 图像改为下一个场景图

整理时间: 2026-05-23

用途: 将 Zealman ComfyUI 工作流 `Y06-图像改为下一个场景图` 接入字字动画, 根据当前图生成“下一场景图”。适合短剧/漫剧里从上一镜头图过渡到下一镜头图。

当前来源:

- 本地 API JSON: `D:\AI-Video-Workflow-Lab\字字工作流插件\可导入字字的API工作流\Y06-图像改为下一个场景图.json`

## 推荐接入路线

使用路线 A: 字字直接连接 ComfyUI。

原因:

- 该工作流已有可直接导入字字动画的 API JSON。
- 这是图像工作流, 在字字的 `图片模型管理 -> ComfyUI工作流` 中导入。
- 输入是一张首帧/参考图, 输出是一张下一场景图。

连接地址应使用 AutoDL `WebUI-6006` 打开的真实 ComfyUI 地址, 不是 Zealman 管理面板 `WebUI-6008`。

## 节点映射

| 节点 | 类型 | 字字中建议设置 | 说明 |
| --- | --- | --- | --- |
| `#12` | `TextEncodeQwenImageEditPlus` | 负向提示词 | 当前默认为空, 用于约束不要出现的内容。 |
| `#19` | `Int` | 图像最长边 | 示例值 `1024`, 控制输入图缩放后的最长边。 |
| `#32` | `LoadImage` | 首帧图片 | 当前图/参考图, 作为生成下一场景的基础。 |
| `#36` | `TextEncodeQwenImageEditPlus` | 正向提示词 | 写下一场景的主体、场景、动作、美学和风格。 |
| `#38` | `SaveImage` | 文件名前缀改为 `comfyui` | 避免复杂路径、中文或日期前缀导致字字取结果不稳定。 |

## 当前 API JSON 中的关键默认值

- `#12 prompt`: 空字符串
- `#19 Number`: `1024`
- `#32 image`: 示例占位图
- `#36 prompt`: 以 `Next Scene:` 开头的下一场景提示词
- `#38 filename_prefix`: 原始值类似 `20260407/image_`, 建议改为 `comfyui`

## 使用流程

1. 确认字字动画已连接 `WebUI-6006` 的 ComfyUI 地址, 且 `/object_info` 和 WebSocket 测试通过。
2. 在字字 `图片模型管理 -> ComfyUI工作流` 中导入 Y06 API JSON。
3. 在素材区选择当前镜头图, 右键设置为 `首帧`。
4. 确认 `#32` 对应首帧图片。
5. 将 `#36` 设置为正向提示词。
6. 将 `#12` 设置为负向提示词。
7. 设置 `#19` 图像最长边, 首轮建议 `1024`。
8. 确认 `#38` 文件名前缀为 `comfyui`。
9. 点击生成图片。

## 正向提示词建议

Y06 的提示词最好明确“下一场景”而不是简单描述当前图。推荐结构:

```text
Next Scene:
主体：保持同一角色身份、服装主特征和脸部一致性。
场景：从室内窗边切换到雨夜街巷, 霓虹灯反射在湿漉漉的地面。
动作：角色回头看向远处, 手中握着信封, 表情紧张。
美学控制：电影感, 中景, 侧逆光, 高对比度, 冷色调, 画面干净。
风格化：短剧分镜图, 写实质感, 人物比例自然, 无文字无水印。
```

负向提示词可写:

```text
文字, 水印, logo, 多余人物, 畸形手, 脸部变形, 低清晰度, 过曝, 模糊, 错乱背景
```

## 参数建议

首轮低风险配置:

- `#19`: `1024`
- 批量数量: `1`

如果后续要给 Wan/LTX 视频工作流当首尾帧, 建议输出比例尽量贴近目标视频比例, 并保持角色脸部和服装稳定。

## 常见问题

### 生成结果不像“下一场景”

提示词里不要只写角色描述, 要明确 `Next Scene` 的场景变化、动作变化和镜头变化。

### 角色不稳定

在正向提示词中加入:

- 保持同一角色身份
- 保持发型、服装、脸部特征一致
- 不改变角色年龄和体型

必要时先用角色多角度/一致性工作流生成更稳定参考图。

### 输出尺寸不合适

Y06 用 `#19` 控制图像最长边, 不是直接设置宽高。需要特定横竖比时, 优先用合适比例的输入图。

### 字字拿不到结果

检查 `#38 SaveImage` 的 `filename_prefix`, 改为 `comfyui`。
