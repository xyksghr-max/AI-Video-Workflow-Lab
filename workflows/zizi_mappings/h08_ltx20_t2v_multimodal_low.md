# H08 字字动画映射卡: LTX2.0 多模态全自动低配

整理时间: 2026-05-23

用途: 将 Zealman ComfyUI 工作流 `H08-文生视频-LTX2.0多模态全自动16秒低配` 接入字字动画, 用文字直接生成短视频。

当前来源:

- 本地 API JSON: `D:\AI-Video-Workflow-Lab\字字工作流插件\可导入字字的API工作流\H08-文生视频-LTX2.0多模态全自动16秒低配.json`

## 推荐接入路线

使用路线 A: 字字直接连接 ComfyUI。

原因:

- 该工作流已有可直接导入字字动画的 API JSON。
- 这是文生视频工作流, 不依赖首帧/尾帧。
- 在字字的 `视频模型管理 -> ComfyUI工作流` 中导入。

连接地址应使用 AutoDL `WebUI-6006` 打开的真实 ComfyUI 地址, 不是 Zealman 管理面板 `WebUI-6008`。

## 节点映射

| 节点 | 类型 | 字字中建议设置 | 说明 |
| --- | --- | --- | --- |
| `#110` | `CLIPTextEncode` | 负向提示词 | 默认包含字幕、模糊、曝光、低质、畸形等负向词。 |
| `#121` | `CLIPTextEncode` | 正向提示词 | 写场景、人物、动作、镜头、台词/旁白。 |
| `#199` | `INTConstant` | 视频宽度 | 示例值 `1024`。 |
| `#200` | `INTConstant` | 视频高度 | 示例值 `600`。 |
| `#218` | `Int` | 视频秒数 | 示例值 `10`; 虽然文件名写 16 秒, 当前 API JSON 默认是 10 秒。 |
| `#220` | `Int` | 内部 FPS | 示例值 `24`; 工作流内部用 `秒数 * FPS + 1` 计算帧数。 |
| `#141` | `VHS_VideoCombine` | 输出文件名前缀保持 `comfyui` | 当前 JSON 已经是 `comfyui`, 输出格式为 `video/h264-mp4`。 |

## 当前 API JSON 中的关键默认值

- `#110`: 负向提示词。
- `#121`: 正向提示词, 示例包含“场景”和“对话”。
- `#199 value`: `1024`
- `#200 value`: `600`
- `#218 Number`: `10`
- `#220 Number`: `24`
- `#141 frame_rate`: `25`
- `#141 filename_prefix`: `comfyui`
- `#141 format`: `video/h264-mp4`
- 关键模型:
  - `ltx-2-19b-dev-fp8.safetensors`
  - `gemma_3_12B_it_fp8_e4m3fn.safetensors`
  - `ltx-2-spatial-upscaler-x2-1.0.safetensors`

## 使用流程

1. 确认字字动画已连接 `WebUI-6006` 的 ComfyUI 地址, 且 `/object_info` 和 WebSocket 测试通过。
2. 在字字 `视频模型管理 -> ComfyUI工作流` 中导入 H08 API JSON。
3. 将 `#121` 设置为正向提示词。
4. 将 `#110` 设置为负向提示词。
5. 设置宽高:
   - `#199`: 视频宽度
   - `#200`: 视频高度
6. 设置时长:
   - `#218`: 视频秒数
   - `#220`: FPS, 首轮保持 24
7. 确认 `#141` 文件名前缀为 `comfyui`。
8. 点击生成视频。

## 正向提示词建议

H08 是文生视频入口, 提示词最好结构化一点:

```text
场景：深秋的芦苇荡，一名蒙面女刺客站在小船上。
镜头：中景镜头缓慢推进，芦苇随风摇动，水面有轻微波纹。
动作：女刺客抬头看向镜头，衣摆轻轻飘动，手握短刀。
情绪：冷静、危险、悬疑。
对话：“风起了，你也该上路了。”
```

建议第一轮少写复杂多段动作, 先验证画面和时长是否稳定。

## 参数建议

首轮低风险配置:

- `#199`: `1024`
- `#200`: `600`
- `#218`: `5` 或 `10`
- `#220`: `24`
- 批量数量: `1`

竖屏短视频可尝试:

- `#199`: `768`
- `#200`: `1280`

如果显存或耗时压力大, 优先降低秒数和分辨率。

## 验收标准

- 字字 ComfyUI 测试通过。
- API JSON 导入后可见节点和可配置字段。
- 正向提示词进入 `#121`。
- 负向提示词进入 `#110`。
- 宽高设置进入 `#199/#200`。
- 秒数设置进入 `#218`。
- 生成结果为 mp4, 字字能拿到输出。

## 常见问题

### 文件名叫 16 秒, 但节点里是 10 秒

以 API JSON 中 `#218` 为准。当前本地文件默认 `#218 Number = 10`, 不是 16。需要 16 秒时手动设置 `#218 = 16`, 但首轮建议先 5-10 秒验证。

### 输出画面比例不对

检查 `#199/#200`。H08 不是首尾帧工作流, 不会自动跟随某张输入图比例。

### 生成太慢或不稳定

优先降低:

- `#218` 秒数
- `#199/#200` 分辨率

### 字字拿不到结果

检查 `#141 VHS_VideoCombine` 的 `filename_prefix` 是否仍为 `comfyui`, 输出格式是否为 `video/h264-mp4`。
