# H27 字字动画映射卡: LTX2.3 首尾帧

整理时间: 2026-05-23

用途: 将 Zealman ComfyUI 工作流 `H27-LTX2.3首尾帧` 接入字字动画, 用首帧和尾帧控制镜头开始与结束, 生成短剧/漫剧镜头视频。

当前来源:

- 手动写词版 API JSON: `D:\AI-Video-Workflow-Lab\字字工作流插件\可导入字字的API工作流\H27-LTX2.3首尾帧.json`
- 自动写词版 API JSON: `D:\AI-Video-Workflow-Lab\字字工作流插件\可导入字字的API工作流\H27-LTX2.3首尾帧-自动写词版.json`

## 推荐接入路线

使用路线 A: 字字直接连接 ComfyUI。

原因:

- 两个版本都已有可直接导入字字动画的 API JSON。
- 这是视频工作流, 在字字的 `视频模型管理 -> ComfyUI工作流` 中导入。
- 需要填 AutoDL `WebUI-6006` 打开的真实 ComfyUI 地址, 不是 Zealman 管理面板 `WebUI-6008`。

## 版本区别

| 版本 | 适合场景 | 正向输入方式 |
| --- | --- | --- |
| 手动写词版 | 已经写好镜头运动、角色动作、场景变化 | 直接修改 `#304 CLIPTextEncode` 为正向提示词 |
| 自动写词版 | 想让工作流根据首尾帧自动写转场描述, 只补台词/旁白 | `#304` 保持不替换, 在 `#376 CR Prompt Text` 写台词或旁白 |

自动写词版的内部逻辑:

1. `#348 llama_cpp_instruct_adv` 分析首尾两张图, 自动生成画面转场描述。
2. `#376 CR Prompt Text` 接收字字输入的台词/旁白。
3. `#373 Text Concatenate` 把自动转场描述和台词/旁白拼接。
4. `#304 CLIPTextEncode` 接收 `#373` 的拼接结果。

所以自动写词版不要把 `#304` 改成正向提示词, 否则会绕开自动写词链路。

## 共同节点映射

两个版本通用:

| 节点 | 类型 | 字字中建议设置 | 说明 |
| --- | --- | --- | --- |
| `#269` | `LoadImage` | 首帧图片 | 输入视频第一帧/起始画面。 |
| `#273` | `SaveVideo` | 文件名前缀改为 `comfyui` | 避免复杂路径、中文或日期前缀导致字字取结果不稳定。 |
| `#315` | `CLIPTextEncode` | 负向提示词 | 默认已有一组英文负向词, 可保留或少量调整。 |
| `#332` | `LoadImage` | 尾帧图片 | 输入视频最后一帧/目标画面。 |
| `#340` | `Int` | 视频边长 | 示例值 `1280`, 控制首尾帧缩放后的参考长边。 |
| `#342` | `Int` | 视频秒数 | 示例值 `10`, 代表 10 秒。 |
| `#343` | `Int` | FPS | 示例值 `24`, 工作流内部用 `秒数 * FPS + 1` 计算视频帧数。 |

文件:

```text
H27-LTX2.3首尾帧.json
```

| 节点 | 类型 | 字字中建议设置 | 说明 |
| --- | --- | --- | --- |
| `#304` | `CLIPTextEncode` | 正向提示词 | 写镜头运动、人物动作、首尾帧之间的变化。 |

手动写词版适合输入完整视频提示词, 例如:

```text
镜头缓慢推进, 女子从门口走向窗边, 衣袖轻摆, 表情从紧张转为坚定, 烛光在脸上轻微闪动, 画面从首帧自然过渡到尾帧。
```

## 自动写词版节点映射

文件:

```text
H27-LTX2.3首尾帧-自动写词版.json
```

| 节点 | 类型 | 字字中建议设置 | 说明 |
| --- | --- | --- | --- |
| `#304` | `CLIPTextEncode` | 不替换 | 该节点接收 `#373` 拼接结果, 不要手动替换。 |
| `#376` | `CR Prompt Text` | 正向提示词 | 写台词、旁白、短剧情绪补充。 |

自动写词版适合在字字提示词栏写台词或旁白, 例如:

```text
女生说：我是个兽医，祖传手艺，专治牛马。
新皇登基那天，一道圣旨直接砸到我家：封我为宁贵妃。
```

提示:

- `#376` 可以写台词、旁白、剧情设定, 不必写完整镜头运动。
- 镜头运动和转场描述由 `#348` 自动根据首尾帧生成。
- 如果自动描述方向不对, 再改用手动写词版。

## 使用流程

1. 在字字动画素材区导入首帧图和尾帧图。
2. 右键起始图片设置为 `首帧`。
3. 右键目标图片设置为 `尾帧`。
4. 导入对应版本 API JSON。
5. 确认 `#269` 对应首帧, `#332` 对应尾帧。
6. 手动写词版: 确认 `#304` 是正向提示词。
7. 自动写词版: 确认 `#304` 是不替换, `#376` 是正向提示词。
8. 确认 `#315` 是负向提示词。
9. 首轮建议:
   - `#342` 秒数: 5-10
   - `#340` 边长: 1024 或 1280
   - `#343` FPS: 24
   - 批量数量: 1
10. 点击生成视频。

## 验收标准

- 字字 ComfyUI 测试通过。
- API JSON 导入后可见节点和可配置字段。
- 首帧和尾帧分别进入 `#269`、`#332`。
- 输出文件名前缀为简单英文, 例如 `comfyui`。
- 生成结果为视频文件, 字字能拿到输出。
- 手动写词版中 `#304` 生效。
- 自动写词版中 `#376` 生效, `#304` 保持连接 `#373`。

## 常见问题

### 自动写词版生成内容不跟台词

检查是否把台词写到了 `#376 CR Prompt Text`。如果写到了 `#304`, 会破坏自动拼接链路。

### 首尾帧没有明显过渡

可能是首尾帧差异太大、提示词太抽象, 或秒数太短。优先尝试:

- 增加秒数到 8-10 秒。
- 手动写词版中明确写“从首帧到尾帧的变化”。
- 减少夸张肢体动作, 使用更平滑的镜头描述。

### 字字拿不到结果

检查 `#273 SaveVideo` 的 `filename_prefix`, 改为 `comfyui`。同时保持 `format` 和 `codec` 为 `auto`。

### 输出太慢或显存压力大

首轮降低:

- `#340` 边长从 1280 降到 1024。
- `#342` 秒数从 10 降到 5。
- 批量数量保持 1。
