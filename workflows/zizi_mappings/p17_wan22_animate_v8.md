# P17 Wan2.2 Animate 动作迁移 V8 映射卡

## 定位

- 工作流: `P17-动作迁移4090-48G显卡专用V8`
- 服务器 API 工作流: `/root/zealman-app/workflows/P17-动作迁移4090-48G显卡专用V8.json`
- 类型: 图片角色参考 + 动作视频驱动
- 推荐用途: 人物跳舞、动作迁移、竖屏短视频动作复刻

该工作流比 `P07-动作迁移-Wan2.2AnimateV4` 更复杂, 使用 Wan2.2 Animate、参考图脸部控制、身体比例映射和可选 Uni3C 控制, 更适合做正式动作迁移模板。

## 关键输入节点

- `299:LoadImage`: 角色参考图
- `275:VHS_LoadVideo`: 动作参考视频
- `300:easy int`: 截取帧数, 30fps 下 `150` 约等于 5 秒
- `264:easy int`: 强制帧率, 建议 `30`
- `294:easy int`: 输出短边长度, 竖屏 9:16 下 `576` 约得到 `576x1024`
- `265:easy float`: 脸部强度, 首轮可用 `1.0-1.2`
- `297:easy float`: 姿态强度, 首轮可用 `1.0`
- `271:easy boolean`: 脸部来源开关
- `370:easy boolean`: 是否使用带 Uni3C 的采样路径
- `278:WanVideoTextEncodeCached`: 正向/负向提示词
- `420:VHS_VideoCombine`: 视频输出, 默认带动作视频音频

## 重要经验

### 脸不像参考图时

优先检查 `271:easy boolean`。

- `false`: 脸部信息可能来自动作视频和参考图混合, 容易把动作视频人物的脸带进去。
- `true`: 使用参考图脸部重复驱动, 更适合角色身份保持。

当目标是“参考图人物跳动作视频动作”时, 建议设置:

```text
271 = true
265 = 1.2
297 = 1.0
```

### P07 与 P17 差异

- P07 更像快捷烟雾测试, 参数少, 速度快。
- P17 更适合正式模板, 可调脸部来源、身体比例、输出尺寸和控制路径。
- 如果 P07 结果动作跟随正常但脸像动作视频人物, 优先改用 P17 并打开参考脸优先。

## 后续优化方向

- 若脸仍不够像: 增加参考图脸部清晰度, 或增加专门的人脸参考/换脸修复后处理。
- 若动作不够像: 尝试 `370 = true` 启用 Uni3C 采样路径。
- 若身体变形: 降低短边至 `512`, 或调整姿态/身体比例映射相关节点。
- 若成本过高: 先用 P07 做动作可行性测试, 再用 P17 做正式出片。
