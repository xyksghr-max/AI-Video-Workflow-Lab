# 分阶段执行方法

每个阶段都按「目的、操作、验收标准」推进。

## Phase 0: 本地项目控制台

目的: 建立长期协作的目录、文档、Git 忽略规则和工作流索引。

操作:
- 创建项目目录骨架。
- 写入部署和工作流规划文档。
- 初始化 Git 仓库。
- 忽略模型、输出、缓存、日志和密钥。

验收标准:
- 本地仓库可提交轻量文档。
- 大文件和密钥不会被默认提交。
- 后续服务器部署有清晰检查清单。

## Phase 1: AutoDL 服务器体检

目的: 确认 Zealman ComfyUI v8.51 的真实目录、磁盘、环境和 GPU 状态。

操作:
- 检查系统盘和数据盘挂载。
- 检查 GPU、CUDA、Python、pip/conda。
- 定位 ComfyUI 根目录、custom_nodes、models、output、input。
- 检查 Zealman 应用已带模型和节点。
- 记录服务器路径, 不做清理操作。

验收标准:
- 明确哪些目录可放模型、缓存、输出。
- 明确 ComfyUI 启动方式和访问地址。
- 明确已有节点和模型清单。

## Phase 2: 最小可用视频工作流

目的: 先跑通一个低成本、可复现的视频生成闭环。

操作:
- 选择一个轻量工作流, 优先使用服务器已有模型。
- 准备测试提示词和输入图。
- 运行短时长、小分辨率测试。
- 记录显存占用、耗时、输出路径和失败点。

验收标准:
- 能稳定生成一段测试视频。
- 工作流 JSON、参数、提示词和输出样例有记录。
- 遇到错误能复现并定位。

## Phase 3: 工作流库扩展

目的: 按业务类型沉淀可复用工作流。

操作:
- 分类收集工作流。
- 验证模型来源、节点依赖和许可证。
- 标注适合场景、显存需求、质量等级和风险。
- 把提示词模板和 API 替换点写清楚。

验收标准:
- 每个工作流都有用途、依赖、运行步骤和验收样例。
- 可以按业务目标快速选择工作流。

## Phase 4: 生产级管线

目的: 把故事、分镜、图像、视频、声音、字幕和后期组合成稳定生产流程。

操作:
- 建立角色卡和世界观模板。
- 建立镜头表和提示词模板。
- 接入图片、TTS、字幕、翻译和提示词扩写 API。
- 形成可复用批处理流程。

验收标准:
- 从剧本到成片有明确步骤。
- 关键节点可复现、可替换、可回滚。
