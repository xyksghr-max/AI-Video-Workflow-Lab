# 存储、安全和 Git 规则

## 可以进入 Git 的内容

- 规划文档
- 工作流说明和轻量 JSON
- 提示词模板
- 配置模板
- 脚本
- 资产清单
- 复盘记录

## 不进入 Git 的内容

- 模型权重
- 生成视频和音频
- 大图片素材
- 缓存
- 日志
- API key
- 服务器密码
- 私人素材原文件

## 本地目录原则

- `models/`: 本地模型占位, 内容默认忽略。
- `outputs/`: 本地输出占位, 内容默认忽略。
- `cache/`: 缓存占位, 内容默认忽略。
- `logs/`: 日志占位, 内容默认忽略。
- `assets/`: 只提交说明和清单, 大媒体文件默认忽略。

## 服务器目录原则

服务器上必须先确认磁盘情况, 再决定:

- 模型目录
- 输出目录
- 缓存目录
- ComfyUI 临时目录
- 下载暂存目录

不能把大模型、输出和缓存放系统盘。路径以实际服务器检查结果为准。

## 提交规则

推荐提交节奏:

1. 规划或文档变化单独提交。
2. 一个工作流验证完成后单独提交。
3. 服务器部署记录单独提交。
4. API 集成单独提交。

提交前检查:

- 没有 `.env` 或密钥。
- 没有模型权重。
- 没有输出视频、音频或大图。
- 没有无关临时文件。
