# AutoDL + Zealman ComfyUI 部署计划

## 基本策略

- 服务器运行生成任务, 本地仓库保存规划、索引、配置模板和复盘记录。
- 拿到服务器后先体检, 不直接下载大模型。
- 大文件下载必须进入 `screen` 会话。
- 模型、输出、缓存放数据盘或 AutoDL 推荐的持久化目录, 不放系统盘。
- 任何最新教程、模型地址、节点仓库和依赖版本, 执行前先联网核验。

## 当前计划使用的实例

- AutoDL 应用: zealman-ComfyUI v8.51
- 区域: 西北B区
- GPU: 4090-48G
- 计费: 按量计费, 截图参考价约 ¥2.88/小时/每 GPU
- 资源范围: vCPU 12~22 核, 内存 64~100 GB

初步判断: 这个配置适合作为第一台验证服务器。48G 显存对 ComfyUI 视频工作流比较友好, 可以先做 Zealman 应用体检、已有工作流盘点和小分辨率图生视频/首尾帧视频测试。正式长视频、批量任务或更重模型, 等第一轮体检后再决定是否需要 96G 或 H800。

## 第一批 48G 显存验证目标

以下模型和能力作为 4090-48G 的首批候选验证目标。先按小分辨率、短帧数、低步数跑通, 再逐步提高质量参数。是否已安装、具体下载源、节点版本和工作流兼容性, 以服务器体检和实时核验结果为准。

### Wan2.2 系列

- Wan2.2-I2V-A14B FP8
- Wan2.2-Animate-14B FP8
- Wan2.2-T2V-A14B FP8
- Wan2.2 Fun Control FP8
- Wan2.2 Fun Inp / FLF2V FP8

### LTX-2.3 系列

- LTX-2.3 22B dev FP8
- `ltx-2.3-22b-dev-fp8.safetensors`
- `gemma_3_12B_it_fp4_mixed.safetensors`
- `ltx-2.3-spatial-upscaler-x2-1.0.safetensors`

### 建议验证顺序

1. 先检查 Zealman 应用是否已经内置相关工作流、节点和模型。
2. 先跑 Wan2.2 I2V/T2V 的极小测试, 验证 ComfyUI 基础视频链路。
3. 再跑 Animate、Fun Control、Fun Inp / FLF2V, 验证动作、控制和首尾帧类能力。
4. 最后验证 LTX-2.3 22B dev FP8 和空间超分, 记录显存、速度和质量上限。
5. 通过后再决定是否下载缺失模型或扩展到批量生产参数。

## 初次登录后的检查项

1. 磁盘:
   - 查看系统盘、数据盘、临时盘、挂载点和剩余空间。
   - 确认哪个目录适合放模型、输出和缓存。
   - AutoDL 常见候选路径可能包括 `/root/autodl-tmp`, 但必须以实际 `df -h` 结果为准。

2. GPU:
   - 查看 GPU 型号、显存、驱动和 CUDA。
   - 记录空闲显存和已有进程。

3. Zealman ComfyUI:
   - 定位 ComfyUI 根目录。
   - 定位 `custom_nodes`, `models`, `input`, `output`, `temp`。
   - 确认启动脚本和端口。
   - 检查已安装节点和已存在模型。

4. Python 环境:
   - 确认使用 conda 还是 venv。
   - 确认 pip 源和下载速度。
   - 必要时再切换清华源等镜像。

5. 网络:
   - 测试 Hugging Face、GitHub、ModelScope、阿里云盘或其他模型源可达性。
   - 慢速源优先寻找官方镜像或可信镜像。

## 大文件下载规范

下载前:
- 确认模型是否真的被当前工作流需要。
- 确认许可证和来源。
- 确认目标目录位于数据盘。
- 确认磁盘空间足够。

下载中:
- 使用 `screen`。
- 给会话起可读名字, 例如 `download-wan22-i2v`。
- 保存下载命令和来源链接。

下载后:
- 校验文件大小, 必要时校验 hash。
- 记录模型路径、用途、来源、下载日期。
- 不把模型文件加入 Git。

## 第一台服务器验收标准

- 明确服务器目录布局。
- ComfyUI 可启动并访问。
- 能看到 Zealman 应用自带工作流、节点和模型。
- 能用已有资源跑一个极小测试任务。
- 本地文档记录服务器检查结果。
