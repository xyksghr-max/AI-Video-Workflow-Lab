# 工作流库方法

## 分类

当前工作流库按业务目标分类:

- 短剧视频: 角色一致性、镜头连续性、首尾帧、分镜到视频。
- 漫剧视频: 漫画分镜、2.5D 运镜、口型、台词字幕。
- 真人短剧: 真人参考、换装、换脸、动作迁移、口型和高清修复。
- 二次元二创: 角色保持、风格迁移、参考图生视频。
- 影视二创: 剪辑识别、翻译、配音、字幕、修复、重剪。
- 转场/接力: 大跨度场景过渡、提示词接力、镜头衔接。
- 修复/高清: 插帧、补帧、超分、去噪、HDR、调色。

## 每个工作流的记录字段

每个工作流都应记录:

- 名称
- 适合业务
- 输入类型: 文本、图片、首尾帧、视频、音频、姿态、深度等
- 输出类型
- 推荐模型
- 必需节点
- 可选节点
- 显存需求
- 推荐分辨率和时长
- 速度估计
- 优点
- 缺点
- 常见报错
- API 可替换点
- 提示词模板
- 验收样例

## 筛选优先级

1. 能稳定跑通。
2. 依赖清楚, 模型来源可信。
3. 适合当前业务方向。
4. 显存和时间成本可控。
5. 能和外部 API 组合。
6. 有可复用参数和提示词模板。

## 首批建议验证方向

1. 图生视频/首尾帧视频: 最适合短剧和漫剧的镜头生产。
2. 角色一致性生图: 先锁角色资产, 再进入视频。
3. 口型和 TTS: 为真人短剧、漫剧和出海配音打基础。
4. 高清修复和插帧: 提升可发布质量。
5. 剪辑识别和节奏分析: 用于二创和拉片。
